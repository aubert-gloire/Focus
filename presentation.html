Problem Statement
Existing focus tools block websites by domain. They cannot distinguish between a student using YouTube for a tutorial versus procrastinating on gaming videos. This makes them blunt instruments that either over-block or under-block depending on context.
FOCUS makes tab blocking context-aware by classifying every tab against the user's declared task using on-device machine learning.
How It Works

Three-layer decision engine

1
Hard rules (instant)
Task domain → always allow. Known social platforms (Instagram, TikTok, Twitter…) → always block. Handles ~60% of cases before ML runs.
2
ML classifier (on-device)
Logistic regression on 5 semantic features. Runs in <1ms per tab switch. No network call. Handles the grey zone.
3
TF-IDF fallback
Cosine similarity between task description and page title as a final relevance signal.

Features

Built to compete with paid tools

Focus control

  • Strict mode — no overrides, no escape
  • Distraction budget — set max overrides per session
  • Scheduled start — delay session by N minutes
  • Scans all open tabs on session start

Visibility & feedback

  • ML confidence score on every blocked page
  • Live decision log — last 5 tab switches with probability
  • Session summary with focus score + grade (S/A/B/C)
  • Daily streak tracking

Analytics dashboard

  • 7-day focus score bar chart
  • All-time stats (sessions, hours, avg score)
  • Top distracting domains visualization
  • Recent sessions with letter grades

Privacy first

  • Everything runs locally — no server, no account
  • All storage in chrome.storage.local only
  • ML weights bundled in extension — no API calls
  • Session data never leaves the device
Technical Architecture

Chrome MV3 extension — fully local

Extension layer

  • Manifest V3 service worker
  • chrome.tabs API — tab switch detection
  • chrome.alarms — scheduled sessions + duration alerts
  • chrome.storage.local — sessions, streak, state
  • chrome.scripting — tab redirect injection

ML layer

  • Python training pipeline (from scratch, no scikit-learn)
  • Batch gradient descent + L2 regularization
  • Weights exported as weights.json (bundled)
  • JS inference: classifier.js + tfidf.js
  • Decision in under 1ms, zero network dependency

UI layer

  • Popup: 3 views (idle / active / summary)
  • Blocked page: countdown ring + confidence bar
  • Dashboard: Canvas bar chart + sessions list
  • Onboarding: first-install walkthrough

Security

  • Full CSP compliance — no inline handlers
  • No external scripts, no CDN dependencies
  • No API keys, no credentials, no secrets
  • Open source — auditable by anyone
